# 遊戲規格書 (Game Specification)：小朋友下樓梯 (網頁版)

## 1. 遊戲概述 (Game Overview)
這是一款基於 HTML5 Canvas 與 JavaScript 的垂直卷軸敏捷生存遊戲。玩家需要控制一名角色在不斷往上升的各種類型階梯間向下跳躍，避免碰到頂部的尖刺天花板或掉入螢幕底部的深淵。

## 2. 技術堆疊 (Tech Stack)
* **核心技術:** HTML, CSS (負責簡單的排版與版面設定), JavaScript (Vanilla JS 負責遊戲邏輯與渲染)
* **渲染方式:** HTML5 `<canvas>` 
* **素材替換:** 專案初期無圖片素材，所有的角色、階梯、陷阱均先以「色塊」實作。

## 3. 核心遊戲機制 (Core Mechanics)

### 3.1 視角與環境
* **畫面尺寸:** 例如 400x600 像素 (直向畫面)，置中顯示。
* **天花板 (Ceiling):** 螢幕最上方有一排固定的尖刺天花板，玩家碰到會持續大量扣血。
* **深淵 (Abyss):** 螢幕最下方為邊界判定區，玩家若掉出螢幕下邊緣，直接 Game Over。
* **垂直卷動:** 角色不隨著視角往下捲動，而是所有的生成物（階梯）以固定的速度不斷向上移動（營造出不斷下樓的視覺感受）。

### 3.2 控制方式 (Controls)
* 玩家僅能控制角色的水平移動（左/右）。
* **鍵盤控制:** 左方向鍵 (`ArrowLeft`) 往左移動，右方向鍵 (`ArrowRight`) 往右移動。

### 3.3 角色屬性 (Player)
* **外觀:** 紅色正方形 (例如 24x24 px)。
* **行為:**
  * 自由落體：當沒有站在階梯上時，受到重力影響向下掉落。
  * 平台跟隨：當準確踩在階梯上時，重力重置，並跟隨階梯的速度向上移動。
* **生命值 (HP):**
  * 初始有一定血量條 (例如 100%)。
  * 碰到天花板或踩到尖刺階梯時扣血。
  * 站在普通階梯上時，可緩慢回復微量生命值。

### 3.4 階梯類型與設計 (Platforms)
所有的階梯統一尺寸 (例如寬度 80px，高度 15px)，由螢幕底端隨機橫向位置生成，並等速往上移動。

1. **普通階梯 (Normal):**
   * **外觀:** 綠色色塊。
   * **效果:** 安全的降落點，站在上面可微幅回復生命值。
2. **尖刺階梯 (Spikes):**
   * **外觀:** 灰色或深紅色色塊 (頂部帶有鋸齒視覺)。
   * **效果:** 玩家落在上面會受到一次性傷害。
3. **滾動階梯/輸送帶 (Conveyor - Left/Right):**
   * **外觀:** 黃色色塊。
   * **效果:** 強制角色往左或往右滑動。
4. **彈簧階梯 (Trampoline):**
   * **外觀:** 藍色色塊。
   * **效果:** 玩家一碰到就會被向上彈飛一段不小的距離。
5. **脆弱/翻轉階梯 (Fragile):**
   * **外觀:** 棕色色塊。
   * **效果:** 玩家踩上去後經過短暫延遲 (例如 0.5 秒) 階梯就會消失，讓玩家繼續掉落。

### 3.5 遊戲進度與難度
* **層數系統 (Levels/Depth):** 隨著存活時間的增加，目前的「地下層數」會同步累加。
* **難度提升:** 隨著層數增加，階梯上升的速度會變快，一般階梯數量減少，危險/特殊階梯出現的機率提高。

### 3.6 UI (使用者介面)
* **HUD狀態列:** 畫面頂部顯示目前的 HP 血條與向下層數。
* **開始畫面:** 遊戲標題和「Click to Start」按鈕。
* **結束畫面 (Game Over):** 顯示玩家抵達的最終層數，以及重新開始的按鈕。

## 實作步驟計畫 (Implementation Plan)
1. 建立 `index.html` 基本檔案與 HTML5 Canvas 畫布。
2. 撰寫 `game.js` 建立核心遊戲循環 (Game Loop: Update & Draw)。
3. 實作主角類別 (Player Class)，包含重力、移動與鍵盤偵測邏輯。
4. 實作階梯生成器 (Platform Manager) 即各種階梯的碰撞偵測與特殊效果。
5. 整合頂部尖刺天花板、底部深淵偵測、血量計算以及分數系統。
6. 加入 Start 和 Game Over 的 UI 流程控制。
